## Summary

- Implemented or materially upgraded in this pass: `github`, `gitlab`, `etsy` (kept rich), `eventbrite`, `appletv`, `duolingo`, `adobe` (kept rich), `allen`, `coursera`, `indiatimes`, `vedantu`, `walmart`, `wix`, `vivino` (documented as still shallow), `gravatar`, `libravatar`, `unavatar`, plus public-pivot validators for `foursquare`, `vsco`, `smule`, `bible`, `wanderlog`, and `plex`.
- New cross-cutting behavior added for generated validators: `http_status`, `latency_ms`, and sanitized `proxy_used`.
- Remaining sensitive platforms in this matrix are now present in the registry as structured safety-blocked validators, including blocked-field inventories and explicit skip reasons, until we can confirm a safe, unauthenticated, non-notifying metadata path. `samsclub` is no longer in that blocked set because its pre-login account-options flow is now implemented directly.
- Live-validated email overlays still remain intentionally conservative: modules are only promoted when repeated proxy-backed runs are stable for the curated alias targets.
